## 1. 프로젝트 개요

### 1.1 Core Goal
학생이 알고리즘 동작을 단계별 애니메이션으로 직접 탐색하여,
코드 없이도 실행 흐름을 직관적으로 이해할 수 있게 한다.

### 1.2 배경 및 목적
- **대상 사용자**: 자료구조·알고리즘 수업을 수강하는 대학생
- **Pain Point**: 코드만 보고는 알고리즘이 어떤 순서로 데이터를 바꾸는지 머릿속에서 그림이 그려지지 않는다
- **기대 효과**: 입력값을 직접 바꿔가며 단계별 상태를 눈으로 확인하여, 개념 이해 시간을 단축한다

### 1.3 범위 (Scope)

**In-Scope (v1 포함)**
- 정렬 알고리즘 시각화: 버블 소트, 선택 정렬, 삽입 정렬
- 그래프 탐색 시각화: BFS, DFS
- 입력값 직접 편집 (배열 원소, 그래프 노드/엣지)
- 스텝 제어: 이전 단계, 다음 단계, 자동 재생, 속도 조절
- 각 스텝에서 현재 배열/그래프 상태를 화면에 표시

**Out-of-Scope (v1 제외)**
- 사용자 계정 및 저장 기능 (이유: 핵심 학습 흐름과 무관)
- 코드 에디터 / 직접 알고리즘 작성 기능 (이유: 범위 과대)
- 알고리즘 추가 요청 기능 (이유: 커뮤니티 기능은 v2로)
- 모바일 앱 (iOS/Android 네이티브) (이유: 웹 반응형으로 대체)

## 5. 상세 기능 요건

### 5.1 정렬 알고리즘 시각화 (버블소트·선택정렬·삽입정렬)

**Logic**
1. 사용자가 알고리즘을 선택하고 배열 입력값을 입력한다
2. [시작] 버튼을 누르면 전체 스텝 시퀀스가 미리 계산된다
3. 각 스텝은 현재 비교 중인 원소 쌍을 강조 표시하고, swap 발생 시 위치가 교환되는 애니메이션을 보여준다
4. [이전] / [다음] 버튼으로 스텝을 탐색하거나 [재생]으로 자동 진행한다
5. 정렬 완료 시 "정렬 완료!" 상태를 표시한다

**Validation**
- 배열 원소: 정수만 허용, 최소 2개 ~ 최대 20개
- 원소 값 범위: 1 이상 999 이하
- 공백·문자 입력 시 즉시 입력 거부 (입력창에서 필터)

**Edge Case**
- 데이터 없음 (배열이 비어 있음): "배열을 입력해주세요. 숫자를 쉼표로 구분하여 입력합니다. 예) 5, 3, 8, 1" 안내 표시
- 오류 발생 (스텝 계산 실패): "시각화를 불러오지 못했어요. 다시 시작해볼까요?" + [다시 시작] 버튼

### 5.2 그래프 탐색 시각화 (BFS·DFS)

**Logic**
1. 사용자가 BFS 또는 DFS를 선택하고 시작 노드를 지정한다
2. [시작] 버튼을 누르면 탐색 순서 시퀀스가 미리 계산된다
3. 각 스텝에서 현재 방문 노드가 강조되고, 방문 완료 노드는 색상이 변한다
4. 엣지 탐색 방향이 화살표 애니메이션으로 표시된다
5. 모든 노드 방문 완료 시 탐색 종료 상태를 표시한다

**Validation**
- 노드 수: 최소 2개 ~ 최대 15개
- 시작 노드는 반드시 그래프에 존재하는 노드여야 함
- 사이클 그래프 허용 (방문 처리로 무한루프 방지)

**Edge Case**
- 연결되지 않은 노드 존재 시: "일부 노드는 시작 노드에서 도달할 수 없어요." 안내 표시 (미방문 노드 회색 처리)
- 오류 발생: "그래프를 불러오지 못했어요. 다시 시작해볼까요?" + [다시 시작] 버튼

### 5.3 스텝 제어

**Logic**
1. [이전] 버튼: 현재 스텝이 0이면 비활성화, 아니면 이전 스텝으로 이동
2. [다음] 버튼: 마지막 스텝이면 비활성화, 아니면 다음 스텝으로 이동
3. [재생] 버튼: 현재 스텝부터 마지막까지 자동 진행, 재생 중에는 [일시정지]로 전환
4. 속도 조절 슬라이더: 느리게(0.5×) ~ 빠르게(2×), 기본값 1×
5. 스텝 진행 표시: "스텝 3 / 12" 형태로 현재 위치 표시

**Validation**
- 알고리즘 실행 중(재생 중)에는 입력값 변경 불가 → 버튼 비활성화

**Edge Case**
- 데이터 없음 (시작 전): 스텝 제어 버튼 전체 비활성화 상태
- 오류: 해당 없음 (스텝은 미리 계산된 배열 탐색이므로 런타임 오류 없음)

### 5.4 인터랙션 요약

| 사용자 행동 | 시스템 반응 |
|------------|------------|
| 알고리즘 선택 | 해당 알고리즘용 입력 폼과 기본 예시값 표시 |
| 입력값 수정 후 [시작] | 스텝 시퀀스 재계산 후 첫 스텝 표시 |
| [다음] 클릭 | 다음 스텝 상태로 애니메이션 전환 |
| [재생] 클릭 | 현재 속도로 자동 진행, 버튼이 [일시정지]로 변경 |
| 마지막 스텝 도달 | "완료!" 표시, [처음으로] 버튼 활성화 |

## 6. UI/UX 요건

- **레이아웃**: 단순형 — 상단 알고리즘 선택 + 입력, 중앙 시각화 캔버스, 하단 스텝 제어
- **로딩 상태**: 스텝 계산 중 진행 바 표시 (통상 0.5초 이내)
- **오류 메시지 톤**: 친근하고 가볍게 ("앗, 문제가 생겼어요! 다시 시도해볼까요?")
- **강조 색상**: 비교 중인 원소/노드는 주황색, 확정(정렬 완료/방문 완료)은 초록색, 미방문은 회색
